# Novel2Toon AI — 功能方案（V1 完整版）

> V0 最小验证版见 [FEATURES_V0.md](./FEATURES_V0.md)，先跑通核心链路再升级到本方案。

## 一、产品定位

面向创作者的 **TXT → AI 漫剧视频** 制作工具。用户上传小说文本，系统自动解析剧本、生成分镜画面、配音合成视频。每个阶段支持人工审核和调整。

专业工具感 UI，类似剪映/Figma 的深色主题面板布局。

### 4.3 小说管理面板

| 功能 | 说明 |
|---|---|

#### 4.4.3 剧本 Tab

### 4.5 资产库面板

管理所有视觉资产（角色/场景/道具）的基准图。

#### 4.5.1 角色资产

| 功能 | 说明 |
|---|---|

| 候选图选择 | 网格展示候选图，点击选中一张 |
| 锁定基准图 | 选中后点击「锁定」，后续所有生成严格参照此图 |
| 重新生成 | 不满意可重新生成一批候选图 |
| 锁定状态标识 | 已锁定的角色显示🔒图标，未锁定显示⚠️ |
| 解锁 | 已锁定的基准图可解锁，解锁时选择：保存到资产库 或 移除 |
| 资产库 | 查看该角色历史保存的所有版本，可切换为当前基准图 |
| 解锁影响提示 | 解锁时弹出确认弹窗，显示「将影响 X 集 Y 个镜头」（按 character_id 全局匹配所有集），用户确认后执行标记 |

## 六、并行策略

### 6.1 阶段 1 内部并行

| 任务 | 并行方式 |
|---|---|
| 多集大纲生成 | 各集可并行调用 LLM（共享全局分析结果） |
| 多集剧本生成 | 各集可并行（依赖各自的大纲） |
| 资产提取 | 与大纲生成并行（只依赖分析结果） |

### 6.2 阶段 2 内部并行

| 任务 | 并行方式 |
|---|---|
| 角色资产生成 | 多个角色可并行调用 ComfyUI |
| 场景资产生成 | 多个场景可并行 |
| 道具资产生成 | 多个道具可并行 |
| 分镜帧渲染 | 同一集内：static 类型可并行，action 类型需串行（依赖前一帧） |
| 跨集渲染 | 不同集的分镜可并行 |

### 6.3 并行度控制

- ComfyUI 并发受 GPU 显存限制，设置最大并发数（默认 2）
- LLM 调用设置最大并发数（默认 5）
- Celery worker 数量可配置

## 七、API 设计概要

### 7.1 项目管理

```
POST   /api/projects                    创建项目
GET    /api/projects                    项目列表
GET    /api/projects/{id}               项目详情
PUT    /api/projects/{id}/settings      更新项目设置
DELETE /api/projects/{id}               删除项目
POST   /api/projects/{id}/upload        上传 TXT 文件
```

### 7.2 管线控制

```
POST   /api/projects/{id}/run/{stage}   运行指定阶段
POST   /api/projects/{id}/cancel        取消当前任务
GET    /api/projects/{id}/status        获取管线状态
WS     /ws/projects/{id}/progress       WebSocket 进度推送
```

### 7.3 剧本数据

```
GET    /api/projects/{id}/analysis      获取分析结果
GET    /api/projects/{id}/outlines      获取大纲列表
GET    /api/projects/{id}/scripts/{ep}  获取某集剧本
GET    /api/projects/{id}/storyboard/{ep}  获取某集分镜脚本
```

### 7.4 资产管理

```
GET    /api/projects/{id}/assets/characters                       角色列表
POST   /api/projects/{id}/assets/characters/{char_id}/generate   生成候选图
PUT    /api/projects/{id}/assets/characters/{char_id}/lock        锁定基准图（body: { candidate_file }）
POST   /api/projects/{id}/assets/characters/{char_id}/unlock      解锁（body: { action: "save"|"remove" }）
GET    /api/projects/{id}/assets/characters/{char_id}/library     查看资产库历史版本
PUT    /api/projects/{id}/assets/characters/{char_id}/library/{version}/activate  切换为当前基准图
GET    /api/projects/{id}/assets/scenes                           场景列表
POST   /api/projects/{id}/assets/scenes/{scene_id}/generate
PUT    /api/projects/{id}/assets/scenes/{scene_id}/lock
POST   /api/projects/{id}/assets/scenes/{scene_id}/unlock
GET    /api/projects/{id}/assets/props                            道具列表
POST   /api/projects/{id}/assets/props/{prop_id}/generate
PUT    /api/projects/{id}/assets/props/{prop_id}/lock
POST   /api/projects/{id}/assets/props/{prop_id}/unlock
POST   /api/projects/{id}/assets/generate-all                     批量生成所有资产
POST   /api/projects/{id}/assets/lock-all                         批量锁定
```

### 7.5 分镜管理

```
GET    /api/projects/{id}/frames/{ep}                获取某集所有分镜帧
POST   /api/projects/{id}/frames/{ep}/generate       生成某集所有分镜帧
POST   /api/projects/{id}/frames/{ep}/regenerate     重新生成标记的帧
PUT    /api/projects/{id}/frames/{ep}/{shot_id}/mark 标记某帧需重新生成
GET    /api/projects/{id}/frames/{ep}/{shot_id}/image 获取分镜图片
```

### 7.6 文件服务

```
GET    /api/files/{path}                静态文件服务（图片/音频/视频）
```

## 十、MVP 开发优先级

### P0 — 必须有

1. 项目创建 + TXT 上传（含多卷排序）
2. 章节识别 + 手动调整分集边界
3. 阶段1 全流程：分析 → 大纲 → 剧本 → 分镜脚本

## 十一、页面布局规范

### 11.1 整体布局

```
┌──────────────────────────────────────────────────────────────┐
│  顶部导航栏（48px 固定高度）                                    │
│  [Logo] [项目名称]                    [进度环图标] [设置] [用户] │
└──────────────────────────────────────────────────────────────┘
┌──────────┬───────────────────────────────────────────────────┐
│ 左侧导航  │                                                   │
│ (64px    │           主内容区（自适应宽度）                     │
│  图标栏  │                                                   │
│  或      │                                                   │
│  200px   │                                                   │
│  展开栏) │                                                   │
└──────────┴───────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────────────┐
│  底部状态栏（可折叠）                                           │
└──────────────────────────────────────────────────────────────┘
```

### 11.2 底部状态栏

| 状态 | 说明 |
|---|---|
| 展开态 | 高度约 120px，显示当前任务名称、进度条、日志摘要、暂停/取消按钮 |
| 折叠态 | 收起为 4px 细条，仅保留颜色指示（运行中=蓝色，完成=绿色，错误=红色） |
| 折叠触发 | 点击状态栏顶部的折叠箭头，或双击细条展开 |
| 顶部导航进度环 | 状态栏折叠后，顶部导航栏右侧显示圆环型进度图标：运行中显示动态进度百分比，全部完成显示纯绿色实心圆环 |

### 11.3 最小支持尺寸

| 维度 | 最小值 | 说明 |
|---|---|---|
| 宽度 | 1280px | 低于此宽度显示「请在更宽的窗口中使用」提示横幅，布局不再保证正常 |
| 高度 | 768px | 低于此高度底部状态栏自动折叠，内容区启用内部滚动 |
| 推荐尺寸 | 1440×900px+ | 最佳体验分辨率 |

> 不做移动端适配，专注桌面端专业工具体验。

### 11.4 左侧导航栏

| 状态 | 宽度 | 内容 |
|---|---|---|
| 展开 | 200px | 图标 + 文字标签 |
| 收起 | 64px | 仅图标，hover 显示 tooltip |
| 切换 | 点击底部折叠按钮 | 状态持久化到 localStorage |
